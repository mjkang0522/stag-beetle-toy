## 사용자 테스트 피드백

### 1차 사용자 테스트 - 찬영이

- 사용자 의견: "사슴벌레가 여러 마리였으면 좋겠다."
- 관찰 의미:
    - 현재 사슴벌레 한 마리의 반응뿐 아니라 화면 안에 여러 생물이 함께 존재하는 상황에 관심을 보였다.
    - 프로젝트의 확장 기능 후보로 보관할 가치가 있다.
    - 다만 현재 MVP에 즉시 포함하면 개체별 상태, 터치 대상, 이동, 충돌, 먹이 선택 등 구현 복잡도가 크게 증가한다.

### 랜덤 사슴벌레 방문 루프

- 우선순위: 중간
- 목적:
    - 사용자가 별도의 메뉴나 인벤토리를 관리하지 않아도 여러 색상의 사슴벌레를 만날 수 있게 한다.
    - 만 3세 아이가 설명 없이 관찰하고 터치하는 수동적인 놀이 구조를 유지한다.
    - 찬영이 1차 테스트에서 나온 "사슴벌레가 여러 마리였으면 좋겠다"는 의견을 낮은 구현 복잡도로 반영한다.
- 기본 흐름:
    1. 화면에 알이 자연스럽게 등장한다.
    2. 알을 터치하거나 일정 시간이 지나면 부화한다.
    3. 랜덤 색상의 사슴벌레 한 마리가 등장한다.
    4. 사슴벌레가 약 1~2분 동안 화면에서 활동한다.
    5. 기존 Idle, Walk, Touch, Happy, Drag & Drop, 젤리 먹기 기능을 사용한다.
    6. 활동 시간이 지나면 화면 밖으로 자연스럽게 퇴장한다.
    7. 잠시 빈 화면을 유지한 뒤 새로운 알이 등장한다.
    8. 같은 흐름을 반복한다.
- 퇴장 규칙:
    - Touch, Happy, Eat 상태에서는 퇴장을 보류한다.
    - 사슴벌레를 드래그 중일 때도 퇴장을 보류한다.
    - Idle 또는 Walk 상태가 되었을 때 퇴장을 시작한다.
    - 퇴장 중에는 새로운 Touch, Drag, Follow, Eat 동작을 시작하지 않는다.
    - 초기 퇴장 연출은 기존 Walk를 활용해 화면 밖으로 걸어 나가는 방식으로 검토한다.
- 초기 개발 범위:
    - 한 번에 사슴벌레 한 마리만 출력
    - 기존 단일 개체 구조 유지
    - 랜덤 색상 적용
    - 알 등장 및 부화
    - 체류 시간 관리
    - 자연스러운 퇴장
    - 다음 알 등장 루프
- 초기 범위에서 제외:
    - 여러 마리 동시 출력
    - 인벤토리
    - 개체 선택 및 교체
    - 곤충집 관리
    - 희귀도와 등급
    - 뽑기 재화
    - 중복 개체 처리
    - 다중 개체 상태 관리
- 검토 시점:
    - 기본 상호작용 안정화 이후
    - Capacitor APK 포장 이후
    - 실제 아이 사용 테스트 결과를 보고 구현 여부 결정
- 현재 Sprint와의 관계:
    - 현재 진행 중인 사운드 및 모바일 애니메이션 안정화 작업에는 포함하지 않는다.
    - 이번 항목은 아이디어 백로그로만 관리한다.
    - 현재 코드에 미리 다중 개체 구조를 추가하거나 리팩터링하지 않는다.
- 이후 확장 가능성:
    1. 여러 마리 사슴벌레 확장 검토
    2. 여러 마리 중 먹이를 먹을 개체 선택
    3. 나무 미니씬
    4. 장수풍뎅이 대결
    5. 곤충 도감 검토

각 단계는 한 번에 하나씩 검토한다. 처음부터 완전한 다중 개체 시스템을 구현하지 않는다.
