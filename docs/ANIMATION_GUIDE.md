# 애니메이션 가이드

마지막 업데이트: 2026-07-08

## 기본 규칙

- 사슴벌레의 현재 상태는 `beetle.state`로 관리한다.
- 사슴벌레의 현재 프레임은 `beetle.currentFrame`으로 관리한다.
- 프레임 번호는 1부터 시작한다.
- 상태별 이미지 목록은 `beetleFrames`에 둔다.
- 일회성 애니메이션 규칙은 `animationSettings`에 둔다.
- 화면 반영은 `updateBeetleView()`를 통해 처리한다.
- 새 반응 상태를 시작할 때는 먼저 `clearTimers()`를 호출한다.
- 동시에 두 개 이상의 `setInterval`이 같은 이미지를 바꾸지 않게 한다.

## Idle

- 사용 상태: `idle`
- 사용 애셋: `beetle_idle_01.png`부터 `beetle_idle_03.png`
- 기본 프레임: `01`
- 대기 시간: 1~4초 랜덤
- 대기 후 동작:
  - 45% 확률로 Walk 시작
  - 55% 확률로 Idle Blink 재생

### Idle Blink

- 프레임 수: 3프레임
- 프레임 순서: `02 -> 03 -> 02 -> 01`
- 코드상 순서: `[2, 3, 2, 1]`
- 프레임 지연 시간: 100ms
- 완료 후: 다음 idle 동작 예약

## Walk

- 사용 상태: `walk`
- 사용 애셋: `beetle_walk_01.png`, `beetle_walk_02.png`
- 프레임 수: 2프레임
- 프레임 순서: `01 -> 02 -> 01 -> 02 ...`
- 프레임 지연 시간: 180ms
- 이동 시간: 2.5~4초 랜덤
- X 이동량: 15~30%
- Y 이동량: 5~16%
- X 제한: 15~85%
- Y 제한: 20~80%
- 완료 후: Idle 복귀

원본 사슴벌레 이미지는 왼쪽을 보고 있으므로, 오른쪽으로 이동할 때 `scaleX(-1)`을 적용한다.

## Touch

- 사용 상태: `touch`
- 사용 애셋: `beetle_touch_01.png`부터 `beetle_touch_03.png`
- 프레임 수: 3프레임
- 프레임 순서: `01 -> 02 -> 03 -> 02 -> 01 -> 01 -> 01 -> 01 -> 01`
- 코드상 순서: `[1, 2, 3, 2, 1, 1, 1, 1, 1]`
- 프레임 지연 시간: 100ms
- 시작 조건: 사슴벌레 포인터 입력
- 예외 조건: 현재 상태가 `touch` 또는 `happy`이면 새 touch를 시작하지 않는다.
- 완료 후: Happy 재생

## Happy

- 사용 상태: `happy`
- 사용 애셋: `beetle_happy_01.png`부터 `beetle_happy_03.png`
- 프레임 수: 3프레임
- 프레임 순서: `01 -> 02 -> 03 -> 02 -> 01 -> 02 -> 03 -> 02`
- 코드상 순서: `[1, 2, 3, 2, 1, 2, 3, 2]`
- 프레임 지연 시간: 80ms
- 시작 조건: Touch 완료
- 완료 후: Idle 복귀

## Eat(Open)

- 현재 상태: 애셋만 있음, 코드에는 아직 연결되지 않음
- 후보 상태 이름: `eatOpen`
- 사용 애셋: `beetle_eat_oepn_01.png`부터 `beetle_eat_oepn_04.png`
- 프레임 수: 4프레임
- 권장 방식: 1회 재생
- 권장 완료 후: Eat(Chew) 또는 Happy

주의: 파일명이 `open`이 아니라 `oepn`이다. 실제 파일명을 기준으로 연결해야 한다.

## Eat(Chew)

- 현재 상태: 애셋만 있음, 코드에는 아직 연결되지 않음
- 후보 상태 이름: `eatChew`
- 사용 애셋: `beetle_eat_chew_01.png`, `beetle_eat_chew_02.png`
- 프레임 수: 2프레임
- 권장 방식: 짧은 시간 동안 루프 재생
- 권장 완료 후: Happy 또는 Idle
