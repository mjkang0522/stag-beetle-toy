# 애니메이션 가이드

마지막 업데이트: 2026-07-10

## 기본 규칙

- 사슴벌레의 현재 상태는 `beetle.state`로 관리한다.
- 사슴벌레의 현재 프레임은 `beetle.currentFrame`으로 관리한다.
- 프레임 번호는 1부터 시작한다.
- 상태별 이미지 목록은 `beetleFrames`에 둔다.
- 프레임 순서와 지연 시간 규칙은 `animationSettings`에 둔다.
- 화면 반영은 `updateBeetleView()`를 통해 처리한다.
- 새 반응 상태를 시작할 때는 먼저 `clearTimers()`를 호출한다.
- 동시에 두 개 이상의 `setInterval`이 같은 이미지를 바꾸지 않게 한다.

## Idle

- 사용 상태: `idle`
- 사용 애셋: `beetle_idle_01.png`부터 `beetle_idle_03.png`
- 기본 프레임: `01`
- 대기 시간: 1~4초 랜덤
- 대기 후 동작:
  - 45% 확률로 Walk 시작
  - 55% 확률로 Idle Blink 재생

### Idle Blink

- 프레임 수: 3프레임
- 프레임 순서: `02 -> 03 -> 02 -> 01`
- 코드상 순서: `[2, 3, 2, 1]`
- 프레임 지연 시간: 100ms
- 완료 후: 다음 idle 동작 예약

## Walk

- 사용 상태: `walk`
- 사용 애셋: `beetle_walk_01.png`, `beetle_walk_02.png`
- 프레임 수: 2프레임
- 프레임 순서: `01 -> 02 -> 01 -> 02 ...`
- 프레임 지연 시간: 180ms
- 이동 시간: 2.5~4초 랜덤
- X 이동량: 15~30%
- Y 이동량: 5~16%
- X 제한: 15~85%
- Y 제한: 20~80%
- 완료 후: Idle 복귀

원본 사슴벌레 이미지는 왼쪽을 보고 있으므로, 오른쪽으로 이동할 때 `flipX`를 `-1`로 두고 왼쪽으로 이동할 때 `flipX`를 `1`로 둔다. 아래쪽으로 이동할 때는 `flipY`를 `-1`로 두고, 위쪽으로 이동할 때는 `flipY`를 `1`로 둔다. 화면에는 `translate(-50%, -50%) scale(flipX, flipY)`를 한 번에 적용한다.

## Follow

- 사용 상태: `follow`
- 사용 애셋: `beetle_walk_01.png`, `beetle_walk_02.png`
- 프레임 수: 2프레임
- 프레임 순서: `01 -> 02 -> 01 -> 02 ...`
- 코드상 순서: `[1, 2]`
- 프레임 지연 시간: 180ms
- 시작 조건: 젤리를 드래그해서 놓았을 때
- 목표 위치: 드롭 시점의 젤리 중심 좌표
- 이동 속도: 초당 약 8%
- 도착 판정: 목표 좌표와의 거리 7% 이하
- 완료 후: Eat(Open) 진입

Follow는 Walk 프레임을 재사용하지만, 랜덤 목표가 아니라 젤리의 드롭 위치를 목표로 삼는다.

## Jelly Drag Layer

- 기본 레이어 순서: 배경 -> 젤리 -> 사슴벌레
- 드래그 중 레이어 순서: 배경 -> 사슴벌레 -> 젤리
- 드래그 종료 후: 기본 레이어 순서로 복귀
- 코드 기준: `LAYERS` 상수와 `setJellyDraggingLayer()`가 레이어 값을 관리한다.

젤리 레이어는 y축 위치가 아니라 드래그 상태만 기준으로 바뀐다. 아이가 젤리를 잡고 움직이는 동안에는 잡고 있는 오브젝트가 가장 앞에 보이도록 한다.

## Touch

- 사용 상태: `touch`
- 사용 애셋: `beetle_touch_01.png`부터 `beetle_touch_03.png`
- 프레임 수: 3프레임
- 프레임 순서: `01 -> 02 -> 03 -> 02 -> 01 -> 01 -> 01 -> 01 -> 01`
- 코드상 순서: `[1, 2, 3, 2, 1, 1, 1, 1, 1]`
- 프레임 지연 시간: 100ms
- 시작 조건: 사슴벌레 포인터 입력
- 예외 조건: 현재 상태가 `touch`, `happy`, `eatOpen`, `eatChew`이거나 먹이 주기 루프가 진행 중이면 새 touch를 시작하지 않는다.
- 완료 후: Happy 재생

## Happy

- 사용 상태: `happy`
- 사용 애셋: `beetle_happy_01.png`부터 `beetle_happy_03.png`
- 프레임 수: 3프레임
- 프레임 순서: `01 -> 02 -> 03 -> 02 -> 01 -> 02 -> 03 -> 02`
- 코드상 순서: `[1, 2, 3, 2, 1, 2, 3, 2]`
- 프레임 지연 시간: 80ms
- 시작 조건: Touch 완료 또는 마지막 Eat(Open) 완료 후 빈 접시로 변경
- 완료 후: Touch 흐름에서는 Idle 복귀, 먹이 주기 흐름에서는 빈 접시 깜빡임과 새 젤리 낙하 진행

## Eat(Open)

- 사용 상태: `eatOpen`
- 사용 애셋: `beetle_eat_oepn_01.png`부터 `beetle_eat_oepn_04.png`
- 프레임 수: 4프레임
- 프레임 순서: `01 -> 02 -> 03 -> 04`
- 코드상 순서: `[1, 2, 3, 4]`
- 프레임 지연 시간: 120ms
- 시작 조건: Follow 도착 판정 또는 젤리 근처에서 Follow 시작
- 완료 후: 젤리 이미지를 현재 먹기 단계에 맞게 변경한다. 1, 2번째 단계는 Eat(Chew)로 이어지고 마지막 단계는 Happy로 이어진다.

주의: 파일명이 `open`이 아니라 `oepn`이다. 실제 파일명을 기준으로 연결해야 한다.

## Eat(Chew)

- 사용 상태: `eatChew`
- 사용 애셋: `beetle_eat_chew_01.png`, `beetle_eat_chew_02.png`
- 프레임 수: 2프레임
- 프레임 순서: `01 -> 02`
- 코드상 순서: `[1, 2]`
- 프레임 지연 시간: 160ms
- 재생 방식: 1, 2번째 먹기 단계에서 2프레임 루프를 각각 4회 재생
- 젤리 변경: Eat(Chew) 중에는 바꾸지 않고, 직전 Eat(Open)이 끝난 직후 이미 바뀐 이미지를 유지한다.
- 완료 후: 다음 Eat(Open) 재생

## Feeding Loop

- 시작 조건: Follow 상태에서 젤리와의 거리가 7% 이하가 되었을 때
- 흐름: Follow -> Eat(Open) -> `jelly_eaten_01.png` -> Eat(Chew) 4회 -> Eat(Open) -> `jelly_eaten_02.png` -> Eat(Chew) 4회 -> Eat(Open) -> `jelly_empty_plate.png` -> Happy -> 빈 접시 깜빡임 -> 새 젤리 낙하 -> Idle
- 젤리 변경 타이밍: 각 Eat(Open)이 끝난 직후 변경한다. Eat(Open)이 시작될 때는 바꾸지 않는다.
- 마지막 단계: 빈 접시로 바꾼 뒤 Eat(Chew)를 재생하지 않고 Happy로 넘어간다.
- 빈 접시 처리: `opacity`를 토글해 3회 깜빡인 뒤 `visibility: hidden`으로 숨긴다.
- 새 젤리 처리: `jelly.png`를 화면 위쪽 `top: -12%`에서 기본 위치 `top: 78%`까지 `requestAnimationFrame`으로 떨어뜨린다.
- 입력 제한: 먹는 중과 새 젤리 낙하 중에는 젤리 버튼을 `disabled`로 두고 드래그를 막는다.
- 중복 실행 방지: `feedingLoop.isActive`가 `true`이면 새 먹이 주기 루프를 시작하지 않는다.
